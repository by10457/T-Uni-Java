# 约束

- `core_user` 必须继续作为核心身份表
- 业务用户表必须继续作为扩展表
- 两张表必须共用同一个 `id`
- 先创建 core user，再创建业务用户记录
- 不要让业务用户表自己生成新的主键
- 不要切换成单表设计

## 为什么必须双表

1. **身份与业务解耦**：`core_user` 放核心身份信息（昵称、头像、手机号、禁用/注销状态），业务用户表放该场景的扩展字段（openId、unionId、业务状态等）。这样换业务场景时只需要换业务表，核心身份不用重建。

2. **多端共享 ID**：`core_user.id` 是全局唯一身份标识，admin 和 server 都通过这个 ID 关联用户。如果业务表自增生成不同 ID，admin 侧和 server 侧的 userId 就对不上。

3. **模板可替换性**：`biz_user` 只是模板默认实现。fork 后换成自己的业务用户表（比如 `student`、`merchant`、`patient`），只要保持同 ID 一对一，认证链路不需要改。如果改成单表，换业务时就要重构整条认证链。
